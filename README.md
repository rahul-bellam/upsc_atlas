# UPSC Atlas Explorer

A **global geography study tool with an India-impact lens** for UPSC/PSC revision. Search any city, river, mountain range, protected area, border region, island, or district; inspect it on physical and political maps; then open an AI-generated, exam-oriented geography dossier.

## Features

- **Dual-map interface** — toggle between physical (topographic) and political (boundary/label overlay) basemaps powered by Leaflet and OpenStreetMap.
- **Place search** — Nominatim-powered geocoding returns up to five verifiable matches for any place name.
- **AI geography dossier** — structured synthesis of physical geography, political context, strategic relevance, and India-impact analysis, generated by OpenAI or DeepSeek.
- **Six-month news** — real articles from NewsAPI linked alongside the dossier; AI summarizes only from supplied records.
- **Prelims & Mains angles** — dossier includes map-based facts for Prelims and answer frameworks for Mains.
- **Rate-limited & safe** — configurable Upstash Redis rate limiting (with in-memory fallback), input sanitisation, and no hallucinated coordinates.

## Prerequisites

- Node.js 20.9 – 22.x
- npm
- An OpenAI or DeepSeek API key
- A NewsAPI key with six-month historical-search access (the free tier may not support this)

## Getting Started

```bash
git clone https://github.com/rahul-bellam/upsc_atlas.git
cd upsc_atlas
npm install
cp .env.example .env.local
```

Edit `.env.local` with your API keys, then:

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000).

> Never commit `.env.local` or real credentials. They are ignored by Git.

## Environment Variables

| Variable | Required For | Notes |
| --- | --- | --- |
| `AI_PROVIDER` | AI dossiers | `openai` (default) or `deepseek` |
| `OPENAI_API_KEY` | OpenAI dossier generation | Server-side only |
| `DEEPSEEK_API_KEY` | DeepSeek dossier generation | Server-side only |
| `DEEPSEEK_BASE_URL` | DeepSeek-compatible endpoint | Defaults to `https://api.deepseek.com` |
| `NEWS_KEY` | Six-month place news | Requires historical `/v2/everything` access |
| `NOMINATIM_USER_AGENT` | Public Nominatim geocoding | Include a contact URL or email |
| `UPSTASH_REDIS_REST_URL` | Production rate limiting | Optional; paired with token below |
| `UPSTASH_REDIS_REST_TOKEN` | Production rate limiting | Server-side Upstash Redis REST token |

## Safety & Reliability

- Queries are normalised and limited to 120 characters.
- API routes use Upstash Redis rate limiting when configured, with a bounded in-memory fallback for local development.
- External requests have explicit timeouts and safe failure messages.
- Map coordinates come from Nominatim only — **no AI-invented locations**.
- AI responses must conform to a structured JSON schema before reaching the UI.
- News articles retain original publisher links and publication dates.
- The app clearly distinguishes AI synthesis from sourced articles and background references.

> For horizontally scaled deployments, set `UPSTASH_REDIS_REST_URL` and `UPSTASH_REDIS_REST_TOKEN`. Without them, the app falls back to a per-process limiter suitable only for local dev and low-traffic deployments.
